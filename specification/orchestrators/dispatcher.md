# 数据分发器`DataDispatcher`

数据分发器是把数据任务分发给可用执行单元的组件。它管理任务的排队、调度和分发，以确保任务得到适当的执行。

主要功能包括：

- 任务分发：将待执行的数据任务分发给可用的执行单元，可能涉及到动态资源分配和负载均衡。
- 执行单元管理：管理可用执行单元的状态，监控其负载情况，确保任务分发的合理性。
- 队列管理：管理任务队列，按照优先级和调度算法维护任务的排队顺序。

## 建模方式

Dispatcher 可以使用有向图建模，把数据实体作为 Node、把数据处理器作为 Edge。有两种模式：

1. **有向无环图（DAG）**：作为基础的工作流 Workflow
2. **端到端网络（Network/End-to-End）**：实际上也就是 Dispatcher 默认建模的有向图模式

可以把 Workflow 看做是有向图限制无环的特殊情况，不一定需要单独建模。

> 注：关于 Dispatcher 和 Scheduler 的概念对比，请参考教程中的 [调度概念说明](../../quanttide-tutorial-of-data-engineering/governing/scheduling-concepts.md)。

## 注册机制

在分发或者调度之前，首先需要注册。因此需要定义一套数据实体（主要是数据模型）和处理器的注册机制。

大体可以分为两种情况：

1. **本地注册**：本地 GUI 或者 CLI 读取某个项目文件夹下的程序并提取其中的相关信息到控制面板（Control Panel）中。
2. **云端注册**：类似于微服务引擎的注册机制，把某个云服务（容器或者函数）或者其中的程序注册到 Provider（Data Panel）中，再通过控制面板
